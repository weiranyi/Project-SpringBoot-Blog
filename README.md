# SpringBoot-Blog


### 收获：
- 1、自动化测试：
    - 意义：
        - 不用手工测试，手工测试费时费力，不消耗精力
        - 人总会出错
        - 测试也可以锻炼能
    - 分类
        - 单元测试 unit test（简单快速）
            - 只拿出来很小的模块检查
        - 黑盒测试
            - 只关注接口
            - 使用许多mock
        - 集成测试（可靠、安全、慢、服装）
            - 对整个应用装配好后测试对外接口
        - 冒烟测试
            - 应用开发完成后，做个简单的检查
            - 例如：web项目完成后，简单访问下网页
        - 回归测试
            - 发布新功能，是否破坏了之前的功能
    - 流行度：
        - 1、Junit4
        - 2、Junit5
        - 3、TestNG
    - 实战
        - 命令：mvn test
            - 不需要代码被打包或部署
        - 创建测试类的默认按键为： ctrl+shift+t
